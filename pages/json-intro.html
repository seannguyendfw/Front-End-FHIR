Reading JSON

FHIR sends and receives data in JSON format. It might look intimidating at first — but it's actually very logical and readable once you know the rules.

🤔 First Things First
What is JSON?
📋
JSON is like a super-structured form
Think of a paper intake form at a doctor's office. It has labeled fields: "First Name:", "Date of Birth:", "Insurance Provider:". Each field has a label and a value.

JSON is exactly that — but for computers. It has "keys" (the labels) and "values" (the filled-in answers), structured in a way that any software system in the world can read.

📋 Paper Form

First Name: Sarah
Last Name: Johnson
Date of Birth: 04/23/1985
Gender: Female

📐 The Rules

    
JSON has just 4 simple rules


    

      

        
1️⃣

        
Keys are always in quotes

        
Every label (key) is wrapped in double quotes. This tells JSON "this is a field name, not a value."

        
          "gender": "female"
        
      

      

        
2️⃣

        
Curly braces = one object

        
{} braces wrap a single record or object. Everything inside is the data for that one thing (one patient, one lab result, etc.)

        
          {

            "name": "Sarah"

          }
        
      

      

        
3️⃣

        
Square brackets = a list

        
[] brackets hold a list of multiple things. Patients can have multiple phone numbers, names, or addresses — those go in a list.

        
          "given": ["Sarah", "Marie"]
        
      

      

        
4️⃣

        
Commas separate fields

        
Every key-value pair (except the last one in a block) is followed by a comma. This is just punctuation that separates one field from the next.

        
          "gender": "female",

          "birthDate": "1985-04-23"
        
      

    

  


  
  

    
🔍 Line by Line

    
Let's decode a FHIR Patient together

    
Here is a real FHIR Patient resource, annotated with plain English explanations for every section.


    

      

        

          

            

            Patient Resource
            
          

          

            

          

        

      

      

        

          
📌

          

            
"resourceType": "Patient"

            
Every FHIR resource starts by declaring what type it is. This tells the receiving system "I am a Patient record, use the Patient rules to read me."

          

        

        

          
📌

          

            
"id": "patient-sarah-001"

            
A unique identifier for this specific record in the system. Other resources (like Observations) use this ID to say "this data belongs to patient-sarah-001."

          

        

        

          
📌

          

            
"name": [ { ... } ]

            
Name is a list (note the square brackets) because a patient can have multiple names (legal name, preferred name, maiden name). Each name is an object with "family" (last name) and "given" (first + middle names).

          

        

        

          
📌

          

            
"identifier": [ ... ]

            
The patient's MRN (Medical Record Number) or other IDs. This is how real systems identify the patient — not just by name, which isn't unique. MRNs come with a "system" URL that says which organization assigned this ID.
