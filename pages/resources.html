FHIR Resources

FHIR organizes all healthcare data into building blocks called "Resources." Learn the most important ones and what data they contain.

📦 The Concept
What exactly is a "Resource"?
📁
Think of it like file folders in a filing cabinet
Imagine a hospital's paper records room. There are different types of folders: one for patient demographics, one for lab results, one for prescriptions, one for visit notes. Each folder type has a consistent layout — the same fields, in the same order, every time.

FHIR Resources are those standardized folders. A "Patient" resource always contains the same types of fields (name, birthdate, gender, address). A "Observation" resource always contains measurement data (like a blood pressure reading). Consistent. Predictable. Interoperable.
🔑 The Essential Resources
The FHIR resources you'll encounter most

There are ~150 FHIR resources total, but as a BA you'll primarily work with these core ones.

👤

Patient

Demographics

Who the patient is — their identity, demographics, and contact information.

KEY FIELDS:

name birthDate gender address telecom (phone/email) identifier (MRN) maritalStatus language
💡
BA Tip: The Patient resource does NOT contain clinical data. A patient's diagnoses, meds, and lab results are in separate resources that link back to the Patient by ID.
🔬

Observation

Measurements & Labs

Measurements, lab results, vital signs — anything that was measured or observed about the patient.

EXAMPLES:

Blood pressure: 120/80 Heart rate: 72 bpm HbA1c: 6.2% BMI: 24.5 Blood glucose: 95 mg/dL

KEY FIELDS:

code (what was measured) value (the result) unit (mg/dL, bpm, etc.) status effectiveDateTime
🩺

Condition

Diagnoses & Problems

A patient's diagnoses, medical problems, or health conditions — both current and historical.

EXAMPLES:

Type 2 Diabetes (E11) Hypertension (I10) Asthma (J45)
ℹ️
Conditions use standardized code systems like ICD-10 (diagnosis codes) and SNOMED CT to identify the condition in a machine-readable way.
💊

MedicationRequest

Prescriptions

A prescription or order for medication — what drug, what dose, how often, for how long.

KEY FIELDS:

medication (drug name + RxNorm code) dosage (50mg) frequency (twice daily) requester (prescribing doctor) status (active/stopped)
🏥

Encounter

Visits & Interactions

A patient's visit or interaction with the healthcare system — office visit, ER visit, telehealth, inpatient stay.

KEY FIELDS:

class (inpatient/outpatient/emergency) status period (start/end dates) participant (care team) reasonCode
👨‍⚕️

Practitioner

Providers

A healthcare provider — doctor, nurse, therapist, pharmacist. Contains their credentials and contact info.

KEY FIELDS:

name NPI (National Provider ID) qualification (MD, NP, etc.) specialty
🔗 How They Connect
Resources reference each other

Resources don't exist in isolation — they link together to tell the full patient story.

🕸️
References are like hyperlinks between records
A MedicationRequest says "prescribed for Patient/123" — that's a reference to the Patient resource.
An Observation says "result for Patient/123 during Encounter/456" — it references both Patient and Encounter.

This creates a connected web of data, where you can start from any resource and follow the links to get the full picture.

📋 Example: Patient Sarah's Complete Record

👤
Patient/sarah-001
Sarah Johnson, F, DOB 1985
→
🩺
Condition/cond-001
subject: Patient/sarah-001
Type 2 Diabetes
→
💊
MedicationRequest/rx-001
subject: Patient/sarah-001
Metformin 500mg
→
🔬
Observation/obs-001
subject: Patient/sarah-001
HbA1c: 6.8%
