What is this "Bundle"?
When you search for multiple records, FHIR wraps them in a Bundle — a container object. The actual records are inside the entry array. Each entry has a resource field with the actual data.

total = how many matching records exist on the server
entry = the records returned in this response (limited by _count)

🔬 FHIR URL Anatomy
Every FHIR URL follows the same pattern

Understanding the URL structure lets you read — and write — any FHIR query.

https://hapi.fhir.org/baseR4/Patient?_count=5&status=active
Base URL — The server address. Every query starts with this. In real projects, this would be your hospital's EHR endpoint instead of hapi.fhir.org.
Resource type — Which FHIR resource to query. Always capitalized: Patient, Observation, Encounter, etc.
Parameter names — Come after the ?. Parameters that start with _ (underscore) are FHIR system parameters that work on any resource. Others like status are resource-specific.
Parameter values — The filter values. Multiple parameters are joined by &.

Common FHIR search parameters

Parameter Example What it does
_count_count=5Limit how many results to return. Like "show me 5 records."
_sort_sort=-_lastUpdatedSort results. A minus sign (−) means descending (newest first).
_format_format=jsonTell the server to return JSON (vs. XML). Always included.
statusstatus=finalFilter by status. Values differ by resource type (active, final, finished…).
_id_id=592821Fetch one specific record by its FHIR ID. Returns a single resource.
